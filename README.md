# clockin-mcp

Self-hostable hosted MCP-as-a-service. Users sign up, paste their upstream API token (encrypted at rest in SQLite), and MCP clients (Claude Desktop etc.) authenticate via OAuth 2.1 to call tools that proxy to the upstream API on the user's behalf.

## Stack

- React Router 7 (framework mode, SSR)
- better-auth + `@better-auth/oauth-provider` (OAuth 2.1 provider for MCP, JWT)
- Drizzle ORM + SQLite (better-sqlite3)
- Effect (`@effect/platform` HttpClient) for the upstream API client
- `mcp-handler` + `@modelcontextprotocol/sdk`
- AES-256-GCM token-at-rest encryption via `node:crypto`

## Local development

```sh
pnpm install
cp .env.example .env
# fill BETTER_AUTH_SECRET and TOKEN_ENCRYPTION_KEY:
#   openssl rand -base64 32
mkdir -p data
pnpm db:migrate
pnpm dev
```

Open <http://localhost:5173>, sign up at `/sign-up`, paste an upstream API token at `/settings`.

## Connecting an MCP client

The MCP endpoint is `${BETTER_AUTH_URL}/api/mcp/sse`. MCP clients discover OAuth via:

- `/.well-known/oauth-protected-resource/api/mcp`
- `/.well-known/oauth-authorization-server/api/auth`

Test with the inspector:

```sh
npx @modelcontextprotocol/inspector
# then point it at http://localhost:5173/api/mcp/sse
```

Inspector will dynamically register a public client, redirect through `/sign-in` then `/consent`, then call MCP tools with a JWT bearer.

Two built-in tools:

- `echo` — verifies the OAuth + MCP wiring (no upstream call).
- `ping_upstream` — calls `GET ${UPSTREAM_API_BASE_URL}/ping` using the user's decrypted token. Verifies the DB + crypto + Effect + upstream layer end-to-end.

## Environment

| Var | Notes |
| --- | --- |
| `BETTER_AUTH_SECRET` | `openssl rand -base64 32` |
| `BETTER_AUTH_URL` | Public base URL (e.g. `http://localhost:5173`) |
| `DATABASE_URL` | SQLite file path |
| `TOKEN_ENCRYPTION_KEY` | 32 bytes base64. **Treat as permanent** — rotating invalidates all stored tokens. |
| `UPSTREAM_API_BASE_URL` | The API you're proxying |

## Docker

```sh
docker build -t clockin-mcp .
docker run -p 3000:3000 \
  -e BETTER_AUTH_SECRET=... \
  -e BETTER_AUTH_URL=http://localhost:3000 \
  -e TOKEN_ENCRYPTION_KEY=... \
  -e UPSTREAM_API_BASE_URL=https://api.example.com \
  -v clockin-data:/app/data \
  clockin-mcp
```

The entrypoint runs migrations before starting the server.

## Adding new MCP tools

Edit `app/routes/api.mcp.ts`. Inside `createMcpHandler`, call `server.registerTool(name, { description, inputSchema }, handler)`. To call the upstream API, use the pattern from `ping_upstream`: decrypt the user's token, then `runtime.runPromise(effect.pipe(Effect.provideService(UpstreamToken, ...)))`.

Add new HTTP methods to `UpstreamClient` in `app/lib/effect/upstream.ts`.

## Project layout

```
app/
  lib/
    auth.ts                  better-auth server instance
    auth-client.ts           better-auth React client
    auth-server-client.ts    resource-server client (protected-resource metadata)
    env.ts                   zod-validated env
    crypto/vault.ts          AES-256-GCM encrypt/decrypt
    db/
      index.ts               drizzle handle (better-sqlite3, WAL)
      schema.ts              user_token table
      auth-schema.ts         generated by `pnpm auth:generate`
    effect/
      runtime.ts             process-wide ManagedRuntime
      upstream.ts            UpstreamClient + UpstreamToken
  routes/
    sign-in.tsx
    sign-up.tsx
    consent.tsx              OAuth consent screen
    settings.tsx             paste/replace upstream token
    api.auth.ts              mounts better-auth handler
    api.mcp.ts               MCP endpoint
    well-known/              OAuth + OIDC discovery
db/
  drizzle.config.ts
  migrations/
data/                        SQLite file (gitignored)
```

## Regenerating DB schema

After changing `app/lib/auth.ts` plugins:

```sh
pnpm auth:generate   # regenerates app/lib/db/auth-schema.ts
pnpm db:generate     # generates a new SQL migration
pnpm db:migrate
```
